# 内容任务模板：通用网站调研与自动化复盘

## 任务定位

用于任意网站的信息获取、流程探索、后台操作复盘和自动化方案提炼。

这不是具体平台任务，而是内容侧任务模板。

## 适用场景

- 研究一个网站怎么登录/提交/下载
- 观察某个平台的后台流程
- 查找公开资料
- 跑通注册/onboarding 流程
- 分析某个按钮背后的 API
- 把人工试错过程转成自动化脚本

## 输入

```text
任务名称：
目标网站：
起始 URL：
目标：
成功标准：
是否需要登录：
是否允许用户手动处理验证：是/否
输出位置：
```

## 执行策略

### 1. 明确成功标准

先回答：什么叫完成？

例：
- 成功下载文件
- 成功进入后台页面
- 找到某个接口
- 拿到某个公开信息
- 跑通某个流程

### 2. 开始记录

启动 Recorder：
- 记录页面状态
- 记录导航
- 记录网络请求
- 保存截图
- 保存关键文本

### 3. 人类/AI 协同探索

如果用户知道路径，可以由用户手动点。

如果用户不知道路径，可以由 AI 使用 Browser Engine 辅助探索。

遇到验证码、人类验证、登录授权，交给用户处理。

### 4. 复盘

AI 从记录中提炼：
- 成功路径
- 失败路径
- 关键页面
- 关键请求
- 可 API 化步骤
- 必须保留 UI 的步骤
- 稳定选择器
- 异常分支

```md
- ...

## 关键 API 候选

| 方法 | URL | 作用 | 是否可复用 |
|---|---|---|---|

## UI 自动化建议

## API 化建议

## 风险点

## 后续可沉淀为 Site Profile 的内容
```

## 风险边界

- 不绕过验证码
- 不越权访问
- 不采集敏感隐私
- 不高频请求
- 不破坏网站正常服务
